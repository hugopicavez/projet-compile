Passe 3

Le but de la passe 3 est de générer du code pour la machine abstraite.

On a un programme JCas, contextuellement correct, et on cherche à produire un fichier assembleur qui contient le code correspondant à ce programme.

La passe 3 peut ne pas réussir à générer du code ; dans ce cas la compilation doit être interrompue et un message d'erreur s'affiche, avec le numéro de ligne ayant généré l'erreur dans le programme source.

Classes fournies

La classe principale du compilateur JCas est JCasc ("JCas compiler"), dans le paquetage ProjetCompil.Gencode.Src. Dans le répertoire ProjetCompil/Gencode/Test, la commande jcasc fich.cas produit, si la compilation réussit, un fichier assembleur fich.ass dans le même répertoire.

Ce fichier peut être exécuté par l'interprète de machine abstraite : ima fich.ass.

Le paquetage ProjetCompil.Global.Src3 fournit des primitives pour manipuler le programme assembleur généré. Le programme est une suite de lignes. Une ligne comporte une étiquette, une instruction et un commentaire (chacun de ces éléments étant optionnel). Une instruction comporte une opération et 0, 1 ou 2 opérandes. (cf. classes Prog, Ligne, Inst, Etiq, Operation, Operande, NatureOperande).

Travail à effectuer

  • Coder la passe 3 (programmes Java dans Gencode/Src)
  • Ecrire des jeux de tests (dans Gencode/Test)
  • Rédiger une documentation (dans Gencode/Doc) décrivant :
    • les messages d'erreur (limitations du compilateur),
    • l'architecture de la passe 3,
    • les différents algorithmes utilisés en passe 3.

Conseil

Il est important de pouvoir tester efficacement le code écrit. Pour cela, il faut non seulement générer du code pour la machine abstraite, mais également pouvoir tester ce code (avec la commande ima). Il est donc fortement recommandé de commencer par coder les instructions write et new_line, afin de pouvoir exécuter les programmes assembleur générés et vérifier qu'ils affichent ce que l'on attend.

Rendu de la passe 3

Important Respecter impérativement les consignes suivantes :

  • Supprimer toutes les traces (en particulier l'affichage de l'arbre abstrait, du programme décompilé ou du programme assembleur généré).
  • Mettre un fichier read_me.txt dans le répertoire ProjetCompil, avec vos noms, prénoms, login et adresses mail.
  • déposer sur Chamilo l'ensemble de l'arborescence ProjetCompil, sous la forme d'un fichier ProjetCompil.tar.gz.
